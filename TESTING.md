# Сценарии тестирования

## 1. Тестирование моделей (проверка валидации)

### Student
- **Успешное создание** – все поля (FirstName, LastName, Age, DateOfBirth, Email) заполнены корректно → валидация проходит.
- **Ошибка при пустом FirstName** – вызывается сообщение «Поле 'Имя' обязательно для заполнения».
- **Ошибка при неверном формате Email** – сообщение «Введите корректный email адрес».

### Course
- **Успешное создание** – поле Title не пустое → валидация проходит.
- **Ошибка при пустом Title** – сообщение «Поле 'Название' обязательно для заполнения».

### Instructor
- **Успешное создание** – поля FirstName, LastName, Department не пустые.
- **Ошибка при пустом FirstName / LastName / Department** – соответствующее сообщение об обязательном поле.

### Enrollment
- **Успешное создание** – заданы StudentId, CourseId, EnrollmentDate (оценка опциональна).
- **Ошибка при отсутствии StudentId или CourseId** – сообщение «Поле 'Студент' / 'Курс' обязательно для заполнения».

## 2. Тестирование страниц (PageModels)

### Students/Index
- **Загрузка списка студентов** – метод `OnGetAsync` заполняет свойство `Students` данными из БД.  
  *Тест с InMemory:* при наличии тестовых записей список содержит ожидаемое количество элементов.  
  *Тест с Moq:* проверяет вызов `DbSet` и корректное присвоение данных.

## 3. Тестирование контекста базы данных (ApplicationDbContext)

- **Добавление и получение студента** – контекст сохраняет студента, затем находит его по Email.
- **Добавление и получение курса** – аналогично.
